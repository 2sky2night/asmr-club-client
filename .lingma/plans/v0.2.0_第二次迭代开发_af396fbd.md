## 二、播放列表体验优化

### 2.1 搜索功能实现

#### 2.1.1 数据库扩展
**文件**: `lib/services/database_service.dart`
- 新增 `search_history` 表创建逻辑（数据库版本升级到 2）
- 添加方法：
  - `insertSearchHistory(String keyword)` - 插入搜索记录（去重+时间戳更新）
  - `getSearchHistories()` - 获取最近 30 条搜索历史
  - `clearSearchHistories()` - 清空搜索历史

#### 2.1.2 PlayerProvider 扩展
**文件**: `lib/providers/player_provider.dart`
- 新增状态变量：
  - `String _searchKeyword` - 当前搜索关键词
  - `List<Music> _filteredPlaylist` - 过滤后的播放列表
- 新增方法：
  - `void searchPlaylist(String keyword)` - 执行搜索，更新 `_filteredPlaylist`
  - `void clearSearch()` - 清空搜索，恢复完整列表
  - `List<Music> get displayPlaylist` - getter，返回当前应展示的列表（搜索后或原始）

#### 2.1.3 首页搜索 UI
**文件**: `lib/pages/home_page.dart`
- 在 AppBar 下方添加搜索框区域：
  - `TextField` 用于输入搜索关键词
  - 聚焦时显示搜索历史列表（ScrollView，最大高度 200）
  - 搜索历史项点击后自动填入并执行搜索
  - 支持清空搜索按钮（TextField suffix icon）
- 播放列表数据源从 `player.playlist` 改为 `player.displayPlaylist`

#### 2.1.4 搜索历史管理页面
**文件**: `lib/pages/settings_page.dart`
- 在"图片缓存管理"之后新增"搜索历史管理"菜单项
- 点击可清空搜索历史（与缓存管理类似的确认对话框）

### 2.2 滑动到顶部悬浮球

**文件**: `lib/pages/home_page.dart`
- 使用 `ScrollController` 监听滚动位置
- 当滚动偏移量超过一屏高度时，显示右下角悬浮球（`FloatingActionButton` 或自定义按钮）
- 悬浮球出现/消失使用 `AnimatedOpacity` + `AnimatedContainer` 实现过渡动画
- 点击悬浮球时，使用 `scrollController.animateTo()` 平滑滚动到顶部

## 开发顺序

1. **图片缓存**（最简单，独立性强）

   - 测试验证

   - 添加 PopScope
   - 测试验证

   - 数据库扩展

   - 首页搜索 UI
   - 搜索历史管理
   - 测试验证

4. **滚动到顶部悬浮球**（依赖搜索 UI 完成后的布局）
   - 添加 ScrollController
   - 实现悬浮球逻辑
   - 测试验证

## 注意事项

- 所有功能完成后，需要更新版本号到 0.2.0
- 搜索历史的模糊匹配使用 Dart 字符串 `contains` 方法（不区分大小写）
- 悬浮球动画使用 Flutter 内置动画组件，避免引入额外依赖
- 图片缓存管理页面需要处理异步操作（获取缓存大小、清除缓存）
