# 扫描页面 UI 重构方案

## 1. 现状分析
当前 `scan_page.dart` 采用全屏 Loading 遮罩，用户在扫描过程中无法感知进度，容易产生“卡死”的错觉。且原生层目前是批量返回结果，无法实现真正的实时增量更新。

## 2. 优化目标
- **状态可视化**：将页面划分为“配置态”、“运行态”和“结果态”。
- **实时反馈**：在扫描过程中动态展示已发现的音频文件列表。
- **结果汇总**：扫描结束后在同一页面展示成功/失败统计，并提供返回首页的入口。

## 3. 实施步骤

### 任务 1：改造原生层支持流式回调 (MainActivity.kt)
- **引入 EventChannel**：新增一个 `EventChannel` (`com.example.asmr_club_client/scan_progress`)。
- **实时推送**：修改 `scanBilibiliCacheNative` 逻辑，每处理完一个音频文件，就通过 `EventSink` 向 Dart 层发送一条包含路径和元数据的消息。
- **保持兼容**：保留原有的 `MethodChannel` 调用方式作为备选，或将其改为仅返回最终统计数。

### 任务 2：重构 Dart 侧状态管理 (scan_page.dart)
- **定义枚举状态**：创建 `ScanStatus` 枚举（`idle`, `scanning`, `completed`）。
- **监听流数据**：在 `_startScan` 中监听 `EventChannel`，每收到一条数据就更新 `_foundMusics` 列表并触发 `setState`。
- **UI 动态切换**：
  - **Idle**: 显示目录选择和类型下拉框。
  - **Scanning**: 隐藏配置项，显示一个 `ListView` 实时滚动展示已找到的音乐标题，顶部保留一个简单的加载动画。
  - **Completed**: 停止动画，在列表顶部显示统计卡片（成功 N 条，跳过 M 条），底部显示“完成”按钮。

### 任务 3：优化数据库写入逻辑 (media_scanner.dart)
- **异步去重**：由于现在是流式接收数据，建议在扫描全部结束后再统一进行数据库去重和写入，或者在后台静默处理，避免频繁 I/O 影响 UI 刷新频率。

## 4. 预期效果
- **消除恐慌**：用户能看到列表在不断增长，明确感知到程序正在正常运行。
- **交互流畅**：页面内完成所有操作，无需跳转或弹窗，体验更连贯。
- **性能平衡**：虽然 `setState` 频率增加，但通过限制列表最大显示条目（如最近 50 条）或使用 `SliverList` 优化，可确保低端机（如小米 10）依然流畅。
