## 【v0.0.1】需求介绍

这是一个音乐播放软件，能够指定目录进行媒体扫描，扫描完成汇集成播放列表，在播放列表中点击列表项即可进行音乐播放。

### 要求

设计：

- 简单清爽
- 设计因基于行业常见的手机尺寸

编码：

- 风格清晰，需要有注释
- 组件尽量都是用原生组件，降低理解成本

### 数据库模型设计

应用是一个纯前端的应用，但需要持久化数据的能力，这里用sqlite数据库实现。下面是表结构

#### 音乐表（music）

存放了音乐的元数据

| 字段           | 备注                                                         |
| -------------- | ------------------------------------------------------------ |
| 音乐名称       | 字符串                                                       |
| 音乐的播放路径 | 字符串。绝对路径，文件格式：m4s、mp3等。m4s文件格式必须要支持能够播放，是强要求 |
| 音乐的封面图   | 字符串。一个远程资源地址，https协议。字段可以为空            |
| 音乐作者       | 字符串。音乐作者                                             |

### 应用布局

应用底部需要有Tabbar用来切换主页面，主页面包含：

首页、设置页面

首页包含两个组件：一个播放列表和一个音乐播放器

首页布局：播放列表占用主要内容，底部是音乐播放器

##### 播放列表

###### 数据来源

数据来源于：数据库中的音乐表（music）

###### 播放列表展示逻辑

一个展示全部播放列表的页面，列表会占用首页大部分空间。

需要展示：音乐的名称和封面图

异常情况：若当前歌单为空展示空列表，需要有文字等对用户友好提示。

列表项交互逻辑：

1. 单击，立即播放音乐
2. 长按：立即播放音乐；并打开沉浸式播放模式，在下一节“音乐播放器”会描述

更多。展示形式在列表项最右侧，以一个图标作为入口，点击会以菜单的形式展示【v0.0.2】：

- 从播放列表中移除当前音乐，需要二次确认

##### 音乐播放器

###### 音乐播放器的功能

需要支持（音乐播放器的子控件）：

1. 上一首/下一首
2. 播放方式：循环播放、列表播放、列表循环播放
3. 播放实时进度/音乐总时长展示
4. 暂停/播放

###### 展示形态切换

有两种形态：底部展示、全屏展示（沉浸式播放模式）。底部展示为默认样式，这里详细讲全屏展示（沉浸式播放模式），需要以全屏模式展示音乐播放器，具体要求：

1. 全屏展示/底部展示的形态转换需要过渡动画
2. 全屏展示时需要有收起按钮，可以转换成底部展示
3. 布局要求，整体居中展示，突出封面图，下方展示音乐播放器的子控件

#### 媒体扫描页面

##### 表单内容

1. 扫描的目录选择（涉及应用读取本地文件的权限，可能需要用户授权）
2. 音乐类型（决定后续扫描存储音乐元数据的逻辑），可选项：
   1. bilibli缓存视频
   2. 普通音乐（先禁用，本次先不支持）

##### 确认按钮

点击确认：点击确认即可开始扫描，扫描过程中有全屏loading提示，文本提示“媒体扫描中，请勿关闭当前页面”，最好有扫描进度。

- 扫描过程中，不允许用户关闭loading返回上一个页面！
- 扫描完成，弹窗提示，导入了N条音乐，M条音乐导入失败

##### 返回按钮

返回到上一级页面。

##### 音乐类型扫描逻辑处理

扫描时，不记录重复音乐逻辑：若扫描过程中出现的音乐绝对路径，是已经记录在当前数据库音乐表（music）中的了，则跳过并认为是导入成功了。

###### 普通音乐

先禁用，本次先不支持

###### bilibli缓存视频

若选择的目录是此类型，则当前目录是文件夹列表，每个文件夹都存放了音频文件，需要递归查找（一般需要递归三次）

需要查找的文件为：

1. 音频文件：文件一般命名为 audio.m4s
2. 音频元数据：文件一般命名为 entry.json

查找完成后需要解析entry.json来获取音频元数据，此文件是一个json文件，需要解析出相关字段：

- title：音乐名称
- cover：音乐的封面图
- owner_name：音乐作者

最后需要整理为下列的数据结构来存入数据库音乐表（music）中

| 字段           | 写入内容                                      |
| -------------- | --------------------------------------------- |
| 音乐名称       | 解析出的 title                                |
| 音乐的播放路径 | 直接存储audio.m4s文件的绝对路径，用于音乐播放 |
| 音乐的封面图   | 解析出的 cover                                |
| 音乐作者       | 解析出的 owner_name                           |

#### 定时暂停

含义：用户设定时间N分钟后，暂停音乐播放。

点击后会弹出一个弹窗，包含快捷设置。

快捷设置包含了:

- 不开启（关闭定时暂停功能）
- 10分钟后
- 20分钟后
- 30分钟后
- 60分钟后

若当前选择了某个时间需要选中。
